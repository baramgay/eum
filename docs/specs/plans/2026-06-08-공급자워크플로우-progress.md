### 5. XSS 교차검증 — 실제 페이로드로 양방향 실증 (Playwright `window.alert` 후킹)

사전조치: `window.alert`를 `window.__xssFired` 배열에 메시지를 push하는 함수로 후킹(`browser_evaluate`) — 알럿 발생 여부를 실측으로 판별

**(A) 기관→센터 방향**: 기관 담당자 모드에서 다음 페이로드로 데이터 등록
- 제목: `<script>alert('agency-title')</script>제목텍스트`
- 설명: `<img src=x onerror=alert('agency-desc')>설명텍스트입니다XSS검증용`

검증 결과 (`innerHTML` 직접 검사):
- 기관 모드 "내 제출 현황" 목록: `<strong>&lt;script&gt;alert('agency-title')&lt;/script&gt;제목텍스트</strong>` — `hasScriptTag: false`
- 기관 모드 상세 모달: `<h3>&lt;script&gt;alert('agency-title')&lt;/script&gt;제목텍스트</h3>` / `<p>&lt;img src=x onerror=alert('agency-desc')&gt;설명텍스트입니다XSS검증용</p>` — `hasRawScript: false, hasRawImg: false`
- **센터 모드** 검토대기 목록: `<strong>&lt;script&gt;alert('agency-title')&lt;/script&gt;제목텍스트</strong>` — 동일하게 이스케이프
- **센터 모드** 상세 모달: `<h3>&lt;script&gt;alert('agency-title')&lt;/script&gt;제목텍스트</h3>` / `<p>&lt;img src=x onerror=alert('agency-desc')&gt;설명텍스트입니다XSS검증용</p>` — 권한경계(기관→센터)를 넘는 저장형 XSS 완전 차단 확인
- `window.__xssFired: []` — 전 구간에서 알럿 미발생 실증

**(B) 센터→기관 방향**: 센터(컨설팅) 모드에서 위 제출의 상세 모달을 열고 다음 페이로드로 코멘트 작성

검증 결과:
- 센터 모드 코멘트 영역: `<p>- &lt;script&gt;alert('center-comment')&lt;/script&gt;코멘트텍스트&lt;img src=x onerror=alert('center-comment-img')&gt; (2026-06-08 20:45:15)</p>`
- **기관 담당자 모드**로 전환 후 동일 제출 상세 재열람: `<h4>센터 코멘트</h4><p>- &lt;script&gt;alert('center-comment')&lt;/script&gt;코멘트텍스트&lt;img src=x onerror=alert('center-comment-img')&gt; (2026-06-08 20:45:15)</p>` — 권한경계(센터→기관)를 넘는 저장형 XSS도 동일하게 완전 차단 확인
- `window.__xssFired: []` — 코멘트 작성·재열람 전 구간에서 알럿 미발생 실증
