# 센터 담당자 가이드 — 데이터 검토 및 컨설팅

## 목적

데이터 등록·관리 탭의 **센터(컨설팅) 모드**를 사용하여 기관 담당자가 등록한 데이터를
모니터링하고, 품질개선 권고 및 컨설팅 코멘트를 작성하는 방법을 설명한다.

## 전제 조건

- 서버가 실행 중이어야 한다 (포트 5959):
  ```
  cd D:\업무\gd\eum_platform\.worktrees\공급자-워크플로우-impl
  python run.py
  ```
- 브라우저에서 `http://localhost:5959` 접속
- **기관 담당자가 데이터를 최소 1건 이상 등록한 상태**여야 한다
  (기관 담당자 등록 방법은 [agency-submission.md](agency-submission.md) 참고)

---

## 단계별 가이드

### 1단계 — 센터 모드 진입

1. 상단 내비게이션에서 **데이터 등록·관리** 탭을 클릭한다.
2. 모드 토글에서 **센터(컨설팅) 모드** 버튼을 클릭한다.
   - 전체 제출 현황 요약 카드, 평가 영역별 기여 집계, 검토 대기 목록이 표시된다.

---

### 2단계 — 전체 제출 현황 요약 카드 확인

모드 진입 시 상단에 8개 통계 카드가 나타난다:

| 카드 항목 | 설명 |
|-----------|------|
| 전체 제출 수 | 전 기관 제출 합계 |
| 검토 대기 | status = submitted 건수 |
| 승인 | status = approved 건수 |
| 반려 | status = rejected 건수 |
| 품질 통과 | 진단 요약이 "통과"로 끝나는 건수 |
| 품질 개선 필요 | 진단 요약이 있으나 미통과 건수 |
| AI-Ready 추정 | 품질 통과 + 30행 이상 + 설명 20자 이상 동시 충족 건수 |
| 컨설팅 코멘트 누적 | 전 제출에 등록된 코멘트 총 건수 |

카드 수치는 `/api/submission/all` 및 `/api/evaluation/submissions` 응답을 기반으로 렌더된다.

---

### 3단계 — 평가 영역별 기여 집계 확인

"평가 영역별 기여 집계" 카드에서 평가편람 5개 영역의 기여 현황을 막대 그래프로 확인한다:

| 영역 | 기여 판정 기준 |
|------|----------------|
| 개방·활용 | status = approved |
| 품질 | 진단 요약이 "통과"로 끝남 (단, "미통과"는 제외) |
| 분석·활용 | approved + 50행 이상 동시 충족 |
| 공유 | 코멘트 수 > 0 |
| 관리체계 | approved 또는 rejected + 결정메모가 공백이 아닌 경우 |

집계 로직은 `app/evaluation.py` — `compute_submission_contribution(row)`를 서버에서 수행하며,
프런트엔드는 `/api/evaluation/submissions` 응답만 렌더링한다.

---

### 4단계 — 검토 대기 목록 탐색

검토 대기 목록에서 필터·검색·정렬을 활용하여 검토할 제출을 선택한다.

**필터 (상태별)**
- 전체 / 검토대기 / 승인 / 반려 — 버튼 클릭으로 즉시 적용된다.

**검색**
- 검색창에 텍스트를 입력하면 제목, 설명, 기관명, 품질 요약에서 부분 일치 항목만 표시된다.
- 대소문자 구분 없이 동작한다.

**정렬**
- 최신순 — 제출 일시 기준 내림차순 (기본값)
- 품질점수순(오류율 낮은 순) — `quality_summary`에서 오류율을 파싱해 정렬
- 상태순 — 검토대기 → 승인 → 반려 순

목록 각 항목에는 다음 배지가 표시된다:
- 상태 배지 (검토대기 / 승인 / 반려)
- 품질 통과 / 품질 개선 필요
- AI-Ready 후보 (조건 충족 시)
- 즉시 승인 가능 (submitted 상태 + 품질 통과 시)
- 코멘트 N건 (코멘트가 있는 경우)

---

### 5단계 — 제출 상세 확인

목록에서 제출을 클릭하면 상세 모달이 열린다. 모달에서 확인할 수 있는 정보:

**메타데이터**
- 제출 ID, 제출 기관, 제출 일시, 제목, 설명

**배지 요약**
- 상태 / 품질 / AI-Ready / 즉시승인가능 배지 — 목록과 동일한 기준

**AI-Ready 체크리스트**
- 5개 항목별 충족/미충족 여부 및 세부 설명:
  1. 품질진단 통과
  2. 충분한 데이터 규모 (30행 이상)
  3. 메타데이터 충실 (제목·설명·주제·키워드)
  4. 기계가독 형식 (CSV/JSON/Parquet 등)
  5. 표준 라이선스 (CC-BY 등)

**평가 기여도**
- 5개 영역별 기여/대기 상태와 근거 설명 (백엔드 `compute_submission_contribution` 결과)

**온톨로지 연계 후보**
- 메타데이터 키워드 매칭 기반 추천 (`app/ontology.py` — `recommend_ontology_candidates`)

**품질진단 상세**
- 오류율, 기준치, 규칙 수, 오류 건수, 통과 여부
- 규칙별 위반 건수 목록 (NULL/빈값, 중복행 비율, 숫자형 음수 이상치)

**권장 조치**
- 제출 상태·품질·설명 길이를 종합한 권장 행동 문구:
  - "품질진단을 통과했습니다 — 즉시 승인 가능한 개방 후보로 안내해도 좋습니다."
  - "품질 기준 미달입니다 — 품질개선계획 초안을 생성해 기관에 보완을 요청하세요."
  - "설명(메타데이터)이 부족합니다 — 기관에 메타데이터 보완을 요청하세요."
  등

---

### 6단계 — 품질개선 권고 코멘트로 저장

품질개선 권고가 있는 경우 모달에 "품질개선 권고" 섹션이 표시된다:

1. 권고 항목 목록을 확인한다 (예: "NULL/빈값 5건 이상 — 해당 컬럼 데이터 재정제 후 재진단 권장").
2. 권고 섹션 제목 옆의 **코멘트로 저장** 버튼을 클릭한다.
3. 권고 내용이 자동으로 `POST /api/submission/{id}/comment`로 저장되고, 기관 담당자 모드에서도 확인 가능해진다.

---

### 7단계 — 직접 컨설팅 코멘트 작성

모달 하단 textarea에 직접 컨설팅 의견을 작성할 수 있다:

1. textarea에 의견을 입력한다.
2. **코멘트 등록** 버튼을 클릭한다.
3. 코멘트가 즉시 저장되고 모달이 갱신되어 방금 작성한 코멘트가 이력에 표시된다.
4. 전체 현황의 "컨설팅 코멘트 누적" 카드 수치도 갱신된다.

> 기관 담당자는 기관 담당자 모드 상세 모달의 "센터 코멘트" 섹션에서 이 코멘트를 볼 수 있다.

---

### 권장 조치 참고 기준

| 상황 | 권장 행동 |
|------|-----------|
| 품질 통과 + submitted | 즉시 승인 가능 — 기관 담당자에게 승인 안내 |
| 품질 미통과 | 품질개선계획 초안 생성 후 기관에 보완 요청 |
| 설명 10자 미만 | 메타데이터 보완 요청 |
| approved | 후속 활용 현황 모니터링 |
| rejected | 필요 시 기관과 데이터 범위 재협의 안내 |

**품질개선 권고문 생성** 버튼을 클릭하면 해당 제출 기관의 `draft_quality_plan` 결과가
textarea에 채워진다 — 그대로 수정·저장하거나 코멘트 등록에 활용할 수 있다.

## 운영 한계

### 승인 권한 없음

센터 모드에는 승인/반려 버튼이 없다. 이는 설계 원칙에 따른 것이다:
- `openCenterDetail` 함수에 결정 버튼이 존재하지 않고, `openSubmissionDetail`(기관 모드)에만 있다.
- 센터는 컨설팅·모니터링 역할이며, 개방 여부 결정은 **기관 담당자의 자체 결정 권한**이다.

운영 배포 시에도 이 역할 분리는 유지되어야 하며, 필요에 따라 "센터 승인 요청" 워크플로우를
별도 설계해야 한다.

### 인증 미구현

현재 어떤 사용자도 센터 모드에 접근 가능하며, 전 기관의 제출 데이터를 열람할 수 있다.

**운영 배포 시 OAuth2/OIDC 인증 연동 및 역할(기관 담당자 / 센터 담당자) 기반 접근 제어 필요.**

### 실제 LLM 미연동

자연어 질의 탭의 응답은 규칙 기반 폴백(`app/nlquery.py`)으로 동작한다.
실제 LLM(text-to-SQL 모델 등)을 연동하면 자연어 질의 정확도를 크게 향상시킬 수 있다.
센터 모드의 계획서 초안 생성(`app/planning.py`)도 마찬가지로 규칙 기반이며,
LLM 연동 시 더 풍부한 초안 작성이 가능해진다.

**운영 배포 시 공공망 환경 여부에 따라 내부 LLM 서버 또는 외부 API 연동을 검토하라.**
