# 기관 담당자 가이드 — 데이터 등록·관리

## 목적

데이터 등록·관리 탭의 **기관 담당자 모드**를 사용하여 CSV 데이터를 등록하고,
자동 품질진단 결과를 확인한 뒤 자체적으로 개방(승인) 또는 보류(반려)를 결정하는 방법을
설명한다.

## 전제 조건

- 서버가 실행 중이어야 한다 (포트 5959):
  ```
  cd D:\업무\gd\eum_platform\.worktrees\공급자-워크플로우-impl
  python run.py
  ```
- 브라우저에서 `http://localhost:5959` 접속

---

## 단계별 가이드

### 1단계 — 기관 담당자 모드 진입

1. 상단 내비게이션에서 **데이터 등록·관리** 탭을 클릭한다.
2. 탭 내부의 모드 토글에서 **기관 담당자 모드** 버튼을 클릭한다.
   - 화면 좌측에 "새 데이터 등록" 카드, 우측에 "내 제출 현황" 카드가 나타난다.

> 현재 데모 환경에서는 테넌트(기관)가 `48121`(창원시)로 고정되어 있다.
> 인증 미구현에 따른 한계는 아래 [운영 한계](#운영-한계) 항목을 참고하라.

---

### 2단계 — CSV 파일 업로드 및 스키마 확인

1. "새 데이터 등록" 카드에서 **파일 선택** 버튼(`<input type="file" accept=".csv">`)을 클릭하고 등록할 CSV 파일을 선택한다.
2. **업로드 & 미리보기** 버튼을 클릭한다.
3. 업로드가 완료되면 다음과 같은 안내 메시지가 나타난다:
   ```
   스키마 추론 완료: N행, 컬럼 M개. 아래 메타정보를 입력해 등록하세요.
   ```
   - 서버가 pandas dtype을 읽어 컬럼별 DuckDB 타입(VARCHAR / BIGINT / DOUBLE 등)을 자동 추론한다.
   - 스키마 추론 함수: `app/submission.py` — `infer_schema(df)`

---

### 3단계 — 메타정보 입력

스키마 확인 안내가 표시되면 그 아래 메타정보 폼이 나타난다. 각 항목을 입력한다:

| 항목 | 예시 |
|------|------|
| 제목 | 경남 청년 인구 현황 2024 |
| 설명 | 2024년 기준 경남 시군구별 청년(20~39세) 인구 집계 데이터 |
| 주제 | 인구 |
| 키워드 | 청년, 인구, 경남 (쉼표로 구분) |
| 라이선스 | CC-BY |
| 형식 | CSV |

모든 항목은 필수 입력이다.

---

### 4단계 — 등록 및 자동 품질진단

**등록(자동 진단 실행)** 버튼을 클릭한다.

- 서버는 업로드된 DuckDB 테이블에 대해 범용 품질진단(`run_quality_generic`)을 즉시 실행한다.
- 진단 규칙: NULL/빈값 비율, 중복행 비율, 숫자형 음수 이상치 — 3종 기준 5% 초과 시 미통과.
- 등록 완료 후 결과 메시지가 표시된다:
  ```
  등록 완료 — 자동 진단 결과: 규칙 4종 / 오류 0건 / 오류율 0.0% / 통과
  ```

---

### 5단계 — 진단 결과 확인 및 자체 결정

1. "내 제출 현황" 카드에 방금 등록한 항목이 `submitted`(검토대기) 배지로 표시된다.
2. 해당 항목을 클릭하면 상세 모달이 열린다.
3. 모달에서 다음 정보를 확인할 수 있다:
   - 제목, 설명, 품질진단 결과 요약
   - **AI-Ready 체크리스트**: 품질진단 통과 / 충분한 데이터 규모(30행 이상) / 메타데이터 충실(제목·설명·주제·키워드) / 기계가독 형식 / 표준 라이선스 — 5개 항목
   - **평가 기여도**: 평가편람 5개 영역(개방·활용 / 품질 / 분석·활용 / 공유 / 관리체계)별 기여 여부
   - **온톨로지 연계 후보**: 키워드 매칭 기반 추천 객체 유형
4. 진단 결과를 바탕으로 결정을 내린다:
   - **승인(개방)** 버튼 → status가 `approved`로 전환, 개방포털 등록 가능 상태가 됨
   - **반려(보류)** 버튼 → status가 `rejected`로 전환, 보류 상태로 기록됨
5. 결정 메모(선택 사항)를 입력하면 관리체계 평가 기여도에 반영된다.

> 승인/반려 결정은 **기관 담당자만** 수행할 수 있다.
> 센터(컨설팅) 모드에서는 이 버튼이 표시되지 않는다.

---

### 6단계 — 내 제출 목록에서 상태 확인

"내 제출 현황" 카드에서 등록된 데이터의 상태를 확인한다:

| 배지 색상 | 의미 |
|-----------|------|
| 노란색(b-warn) | submitted — 검토대기 |
| 초록색(b-ok) | approved — 승인(개방) |
| 회색(b-na) | rejected — 반려(보류) |

---

### 7단계 — 제출 상세에서 추가 정보 확인

상세 모달의 각 섹션:

- **AI-Ready 체크리스트**: 전체 충족 시 `AI-Ready` 배지 표시
- **온톨로지 연계 후보**: `청년인구`, `사업체`, `청년인프라`, `시군` 등 객체 유형 및 매칭 이유
- **평가 기여도**: 5개 영역별 기여/대기 상태와 근거 설명
- **센터 코멘트**: 센터 담당자가 남긴 컨설팅 의견 이력

---

### 8단계 — 품질개선 권고 시 재등록

진단 결과가 **미통과**이거나 품질개선 권고가 표시된 경우:

1. 권고 내용을 확인한다 (예: "NULL/빈값 5건 — 해당 컬럼 데이터를 채워 재업로드하세요").
2. 원본 데이터를 보완한 뒤 1단계부터 다시 시작하여 재등록한다.

---

### 9단계 — 계획서 초안 생성

"새 데이터 등록" 카드 아래 **계획서 초안 생성** 카드가 있다.

- **공공데이터 개방계획 초안** 버튼: 기관의 제출 현황(개방/보류/검토대기 건수)을 기반으로
  평가편람 양식에 맞는 개방계획 텍스트 초안을 생성한다.
- **품질개선계획 초안** 버튼: 반려 처리된 제출들의 진단 요약을 파싱해 개선과제 목록을 생성한다.
- 생성된 초안은 `<pre>` 영역에 평문으로 표시된다. 복사 후 담당자가 최종 검토하여 사용한다.

> 외부 LLM 없이 규칙 기반으로 생성된다(`app/planning.py`). 공공망(인터넷 차단 환경)에서도 동작한다.

## 운영 한계

현재 구현은 프로토타입 단계이며, 아래 항목은 운영 배포 전에 반드시 보완해야 한다.

### 인증 미구현

현재 로그인 없이 누구나 어떤 테넌트로도 전환 가능하다.
`web/app.js` 326행에 `const TENANT_ID = "48121"; // 데모: 창원시 고정 (로그인 미구현 단계)`
로 명시된 바와 같이, 기관 코드가 소스에 하드코딩되어 있다.

**운영 배포 시 OAuth2/OIDC 인증 연동 필요** — 인증 후 서버에서 `tenant_id`를 세션·토큰에서
읽도록 API와 프런트엔드를 수정해야 한다.

### 멀티테넌트 격리 미구현

기관 간 데이터는 DuckDB 레벨에서 격리되지 않는다.
`tenant_id` 컬럼 필터(`WHERE tenant_id = ?`)에만 의존하며, Row-Level Security나
별도 스키마 분리는 적용되어 있지 않다.

**운영 배포 시 DB 레벨 격리 설계 필요** — 최소한 인증된 `tenant_id`를 서버에서 강제
검증하도록 라우트를 보완해야 한다(현재는 `tenant_id`를 폼 파라미터로 받아 `tenants`
테이블 화이트리스트로만 검증 중).

### 실 DB 및 CDC 미연동

실제 운영 데이터 파이프라인(공공데이터포털 API, CDC 수집기 등) 연결이 없다.
DuckDB 파일 기반 프로토타입으로, 업로드된 CSV 파일만 처리 대상이다.

**운영 배포 시 실제 데이터 파이프라인 연동 필요.**
