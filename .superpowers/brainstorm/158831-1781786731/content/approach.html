온톨로지 고도화 — 방향 선택

현재 시스템: D3 그래프 + 8종 레이아웃 + 5종 분석 + 17개 샘플 데이터(경남). 어떤 방향으로 고도화할까요?

A

데모 시나리오 + 즉시 시각 임팩트

페이지 진입 시 "청년 유출 분석", "공공시설 격차", "사업체 밀도 클러스터" 같은 미리 구성된 시나리오를 선택하면 해당 그래프가 즉시 펼쳐지는 방식. 샘플 데이터로 그래프가 자동 레이아웃되고 분석 결과까지 한 화면에 표시됨.

강점

  • 처음 보는 사람도 3초 안에 인상적인 결과
  • 발표·시연 환경에 최적
  • 샘플 데이터 활용도 극대화

약점

  • 시나리오 기획 시간 필요
  • 고정 시나리오 외 탐색은 여전히 직접 해야 함
B

인터랙티브 대시보드 통합

그래프 왼쪽에 시군 KPI 패널을 붙이고, 노드 클릭 시 해당 시군의 도메인별 지표가 카드로 펼쳐지는 구조. 시계열 슬라이더로 연도별 변화를 애니메이션으로 보여주고 순위 변화까지 연동.

강점

  • 탐색성·인터랙션 모두 높음
  • 실제 분석 워크플로우에 가장 가까움
  • 현재 코드베이스 확장으로 가능

약점

  • 레이아웃 복잡도 증가
  • 시계열 데이터 정비 필요
C

A + B 혼합: 시나리오 진입 → 자유 탐색

처음엔 큐레이션된 시나리오 카드(A 방식)로 빠른 인상을 주고, 시나리오 진입 후엔 인터랙티브 대시보드(B 방식)로 자유 탐색 가능하게 전환. "발표용 임팩트 + 실제 사용성" 모두 확보.

강점

  • 첫인상(시연) + 지속 사용성 모두 충족
  • 온보딩 경험 자연스러움

약점

  • A, B보다 구현 범위 넓음
  • 단계별로 나눠서 진행 권장
